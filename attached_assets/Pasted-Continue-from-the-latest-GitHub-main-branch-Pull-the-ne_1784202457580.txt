Currency Detection

Correctly detect:
	•	USD
	•	NGN
	•	EUR
	•	GBP
	•	CAD
	•	AUD
	•	JPY
	•	INR
	•	ZAR
	•	AED
	•	CHF
	•	SEK
	•	NOK
	•	DKK
	•	PLN

Detect and store the actual transaction amount together with its matching currency.

Ignore informational monetary values such as:
	•	currency conversion examples
	•	“equivalent in…”
	•	exchange-rate references
	•	previous balances
	•	estimated totals
	•	promotional prices
	•	“you saved…”
	•	taxes when a final charged total exists
	•	subtotals when a final total exists

Always prioritize the amount representing the final amount actually charged to the customer.

If multiple monetary values appear, determine which one represents the completed transaction and store that amount together with its corresponding currency.

If an email genuinely contains multiple completed purchases (not simply multiple prices for one purchase), extract each completed transaction separately.

Add regression tests covering:
	•	multiple currencies in one email
	•	subtotal vs total
	•	tax vs total
	•	discounts
	•	exchange-rate references
	•	previous balance vs amount paid
	•	multilingual receipts
